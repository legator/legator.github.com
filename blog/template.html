Came across an interesting article on Habra - I got an interview on Facebook. I'm currently looking for work ,

so attention has increased several times. The main idea of ​​the article - the implementation of swirling spiral sequence numbers.

I was wondering if I could solve this problem.

Writed algorithm invented by me and I have not tested or something like that exists . The main thing that I was able to solve it .

The basis of algorithm is to compute the values ​​in angles. Now need to determine how much is left to be cycles depending on the size

of the matrix. Having Written several such matrices found that the number of cycles is rounded value of matrix size ​​divided by 2.

The difference between adjacent angles - one less than the size of matrix side.

To move to the next level - the value increased by one.

for (int i = 0; i < step; i++)

{

matrix[i, i] = start;

start = start + (size - 1) - 2 * i;

matrix[i, size - i - 1] = start;

start = start + (size - 1) - 2 * i;

matrix[size - i - 1, size - i - 1] = start;

start = start + (size - 1) - 2 * i;

matrix[size - i - 1, i] = start;

start = p;

}

The final step - filling in the rest of the matrix - is performed after determining the values ​​in the corner

Example for filling matrix for top lines

int p = start + 1;

for (int j = 1; j < size - 2*i - 1; j++)

{

matrix[i, i + j] = p++;

}
